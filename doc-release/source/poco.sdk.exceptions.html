poco.sdk.exceptions module¶

This module provides several exceptions for poco sdk. These exceptions are only raised in sdk corresponding runtime.

exception NoSuchTargetException[source]¶

Bases: exceptions.Exception

Raises when index out of range on selecting UI element by given index.

exception NodeHasBeenRemovedException(attrName, node)[source]¶

Bases: exceptions.Exception

Raises if node (UI element) refreshed (updated, recycled or destroyed) when retrieving attributes at traversing. For some engines, the UI hierarchy is refreshing in another thread when poco is traversing. At the moment poco is retrieving attribute on the UI element that is refreshed, this may trigger an engine error. Poco sdk catches this engine error and transform into this exception.

exception UnableToSetAttributeException(attrName, node)[source]¶

Bases: exceptions.Exception

Raises when fail to set attributes on UI element. Mostly, the reason may be the UI element is not support to mutate attribute. At the view of sdk implementation, you can throw this exception proactively to indicate that modification is not allowed.

exception NoSuchComparatorException(matchingMethod, matcherName)[source]¶

Bases: exceptions.Exception

When the matcher does not support the given comparison method, this will be raised.
